## 插件系统架构

LabFlow 的插件系统基于 **运行时加载 (Runtime Loading)** 和 **全局依赖注入 (Global Dependency Injection)**。

### 核心概念

1.  **前端组件**: 插件本质上是一个 React 组件，渲染在编辑器的侧边栏中。
2.  **全局暴露**: LabFlow 主应用在 `window` 对象上暴露了核心依赖，包括 `React`, `Lucide` (图标库), 和 `LabFlow` (核心 API)。
3.  **动态注册**: 插件脚本执行时，通过 `window.LabFlow.pluginRegistry.register()` 将自身注册到系统中。

### 运行环境

插件运行在浏览器的沙箱环境中（目前与主应用共享同一个 Window 上下文）。

- **React**: `window.React` (v18+)
- **Icons**: `window.Lucide`
- **Registry**: `window.LabFlow.pluginRegistry`

### 限制

- 插件目前主要用于扩展**侧边栏 (Sidebar)** 功能。
- 插件可以直接读取和插入文档块 (`TypstBlock`)，但不能直接修改编辑器核心渲染逻辑（除非通过修改 Block 数据）。
